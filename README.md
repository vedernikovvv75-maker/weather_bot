# Weather Telegram Bot

Телеграм-бот с погодой OpenWeather: текущая погода, прогноз на 5 дней, расширенные данные по качеству воздуха, сравнение городов и уведомления.

## Структура

- `bot.py` — логика Telegram-бота
- `weather_app.py` — запросы к OpenWeather API + кэш
- `storage.py` — чтение/сохранение данных пользователей
- `User_Data.json` — локальная база пользователей
- `.env.example` — шаблон переменных окружения

## Подготовка

1. Создайте и активируйте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` и заполните значениями из `.env.example`.

## Переменные окружения

```env
OW_API_KEY=your_openweather_key
BOT_TOKEN=your_telegram_token
```

## Запуск

```bash
python bot.py
```

## Хранилище пользователя

Формат данных в `User_Data.json`:

```json
{
  "123456789": {
    "city": "Москва",
    "lat": 55.7558,
    "lon": 37.6176,
    "notifications": {
      "enabled": true,
      "interval_h": 2
    }
  }
}
```

Дополнительно может храниться `last_sent_at` для имитации планировщика уведомлений.
# Weather Telegram Bot

Telegram-бот для получения текущей погоды, прогноза на 5 дней, анализа качества воздуха и уведомлений.

## Установка

1. Создайте виртуальное окружение и активируйте его.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Создайте `.env` по образцу `.env.example` и укажите реальные токены:
   - `OW_API_KEY`
   - `BOT_TOKEN`

## Запуск

```bash
python bot.py
```

## Возможности

- `/start` и меню с кнопками
- Текущая погода (город/геолокация)
- Прогноз на 5 дней с выбором дня через inline-кнопки
- Сравнение двух городов
- Расширенные данные (погода + качество воздуха)
- Уведомления с интервалом в часах
- Inline-режим по названию города

## Хранилище данных

Пользовательские данные сохраняются в `User_Data.json`:

```json
{
  "123456789": {
    "city": "Москва",
    "lat": 55.7558,
    "lon": 37.6176,
    "notifications": {
      "enabled": true,
      "interval_h": 2
    }
  }
}
```

Для уведомлений хранится дополнительное поле `last_sent_at` (unix-время последней отправки).
